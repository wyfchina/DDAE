# Adaptive S&OP / DDS&OP 本地启动操作说明

## 什么时候需要启动服务

系统重启、电脑关机后重新开机、或手动关闭本地服务后，都需要重新启动 Web 服务。

应用地址：

```text
http://127.0.0.1:5188
```

## 1. 打开 PowerShell

进入项目目录：

```powershell
cd "D:\Documents\DDAE"
```

## 2. 启动应用服务

执行：

```powershell
& "$env:USERPROFILE\.dotnet-sdk-9\dotnet.exe" run --project src\AdaptiveSopDdsop.Web\AdaptiveSopDdsop.Web.csproj --urls http://127.0.0.1:5188
```

看到类似下面的信息，表示服务已经启动：

```text
Now listening on: http://127.0.0.1:5188
Application started. Press Ctrl+C to shut down.
```

然后在浏览器打开：

```text
http://127.0.0.1:5188
```

## 3. 验证服务是否正常

可以在另一个 PowerShell 窗口执行：

```powershell
Invoke-WebRequest http://127.0.0.1:5188 -UseBasicParsing
```

如果返回 `StatusCode : 200`，说明页面服务正常。

也可以验证数据接口：

```powershell
Invoke-RestMethod http://127.0.0.1:5188/api/validation-data
```

正常情况下会返回产品族、SKU、36 个月展望期、AS&OP 流程、DDS&OP 元素等数据。

## 4. 停止服务

如果服务是在当前 PowerShell 窗口启动的，按：

```text
Ctrl+C
```

即可停止。

如果端口被占用，可以执行：

```powershell
$conn = Get-NetTCPConnection -LocalPort 5188 -State Listen -ErrorAction SilentlyContinue | Select-Object -First 1
if ($conn) { Stop-Process -Id $conn.OwningProcess -Force }
```

## 5. 修改代码后重新验证

先停止正在运行的服务，然后执行：

```powershell
& "$env:USERPROFILE\.dotnet-sdk-9\dotnet.exe" run --project tests\AdaptiveSopDdsop.Tests\AdaptiveSopDdsop.Tests.csproj
& "$env:USERPROFILE\.dotnet-sdk-9\dotnet.exe" build src\AdaptiveSopDdsop.Web\AdaptiveSopDdsop.Web.csproj
```

当前应看到：

```text
19 test(s) passed.
已成功生成。
0 个警告
0 个错误
```

然后重新启动应用服务。

## 6. 常见问题

### 端口 5188 被占用

先执行停止服务命令，再重新启动。

### build 时提示 exe 或 cache 文件被占用

通常是因为 Web 服务还在运行。先停止服务，再执行 build。

### 浏览器打不开页面

确认 PowerShell 里服务仍在运行，并确认访问地址是：

```text
http://127.0.0.1:5188
```

不要使用旧的 `file:///.../index.html` 地址。
