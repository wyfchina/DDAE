**结论先说清楚：有用 routing，有设定库存，也能给控制点 / 库存缓冲点 / 时间缓冲点建议，但三者成熟度不同。**

1. **Routing 是否用到**
有。当前不是只在界面展示 routing，而是实际参与 RCCP、资源负荷、产品族资源缺口计算。

当前 routing 是“SKU 对关键资源的单位能力消耗”，例如：

| 产品族 | 资源 | 单位资源消耗 |
|---|---:|---:|
| 星载电子 | `RES-HARNESS` | 0.52 |
| 星载电子 | `RES-TVAC` | 0.20 |
| 有效载荷 | `RES-CLEAN` | 0.70 |
| 有效载荷 | `RES-TVAC` | 0.42 |
| 热控结构 | `RES-AIT` | 0.30 |
| 热控结构 | `RES-HARNESS` | 0.18 |
| 卫星平台 | `RES-AIT` | 0.56 |
| 卫星平台 | `RES-TVAC` | 0.24 |

计算逻辑是：

```text
资源需求负荷 = 预计补货订单数量 × routing 单位资源消耗
负荷率 = 资源需求负荷 / 可用能力
```

所以你看到的“资源负荷 201.8”这类值，本质上是若干 SKU 的补货订单通过 routing 折算出来的资源需求，不是凭空写死的数字。

目前 routing 粒度还是 **DDS&OP 周度粗能力 routing**，不是 SDBR / DDOM 里的详细工序、换型、班次、维护、假期、日历排程。这个分工是对的：DDAE 看中期能力压力，SDBR 负责详细排程。

2. **库存是否设定**
有。当前 seed data 里设定了每个 SKU 的库存位置，包括：

```text
On Hand 现有库存
Open Supply 已开放供应
Qualified Demand 合格需求
```

然后系统用 DDMRP 口径计算：

```text
净流动量 Net Flow = 现有库存 + 已开放供应 - 合格需求
```

之后每周根据需求消耗、红黄绿缓冲区、订货周期、MOQ、提前建库等规则，推演未来库存水位和补货订单。

所以库存参与了这些结果：

- SKU 净流动量趋势
- 红 / 黄 / 绿区状态
- 是否触发补货
- 补货数量
- 平均库存金额
- 峰值库存金额
- 产品族库存风险
- 主设置治理里的库存缓冲建议

但要注意：现在库存是演示 / 测试数据，不是从 ERP、WMS 或真实 DDOM 执行系统接入的生产库存。

3. **能否给控制点、库存缓冲点、时间缓冲点建议**
可以，但成熟度不同。

**库存缓冲点建议：当前最完整。**

系统已经能根据场景预览结果、红区 SKU、MOQ / 订货周期覆盖、提前建库、供应缺口，生成库存缓冲相关建议。例如：

| 建议对象 | 建议原因 |
|---|---|
| `AV-FPGA-203` 进口空间级 FPGA 板 | 长交期、供应商能力受限、星载电子关键件，适合作为库存缓冲重点 |
| `CBL-HAR-402` 星上电缆束套件 | ADU 高、会消耗线束资源，适合作为流动性保护缓冲 |
| `PAY-EO-101` / `PAY-SAR-102` 有效载荷 | 高价值件，建议谨慎缓冲，可更多依赖时间缓冲和产能保护 |

**时间缓冲点建议：已有第一版规则型建议。**

例如当前案例里，`Microchip Space / 进口空间级 FPGA` 已有类似建议：

```text
当前保护窗口：8 周
建议保护窗口：12 周 + Act/Late 阈值
原因：出口许可延迟、长交期、供应承诺能力接近上限
```

当系统发现供应缺口时，也会建议：

```text
在缺口周之前增加供应保护提前期
增加 Act/Late 阈值
提前协调供应商承诺能力
必要时启用替代料或管理取舍
```

但这还是规则型，不是完整的供应提前期概率分布仿真。

**控制点 / 解耦点建议：当前可以展示和治理，但自动推荐还比较初级。**

系统里已有 `Decoupling Point` 主设置，也能在主设置治理中保存“设置某产品族半成品 Buffer”这类建议。比如：

```text
星载电子半成品解耦点
建议设置星载电子半成品 Buffer
触发原因：星座批产长期大单
```

我建议当前案例中优先关注这些控制点：

| 控制点建议 | 原因 |
|---|---|
| 星载电子半成品 / FPGA 板级件 | 进口 FPGA 长交期，供应风险高，且影响多个星载电子 SKU |
| 有效载荷装配前缓冲点 | 有效载荷消耗洁净装配和热真空资源，价值高、周期长 |
| 星上电缆束套件缓冲点 | 高 ADU，会形成线束资源压力，适合保护 AIT 节奏 |
| 热真空试验前时间缓冲点 | `RES-TVAC` 容易形成中期能力瓶颈，应保护试验窗口 |

不过这里要加一句边界：**当前系统还没有完整的网络结构自动选点算法**。也就是说，它能根据异常、场景结果和主设置规则提出建议，但还没有基于 BOM 层级、累计提前期、变异性、资源约束、供应风险进行自动解耦点优化。这个可以作为后续增强方向。

整体判断：  
现在 DDAE 的逻辑已经从“看图表”进入了“用 routing + 库存 + DDMRP 参数推演场景”的阶段。下一步如果要更接近产品级方法学，就是把控制点建议从规则型提升为“网络结构评分 + 场景验证 + 主设置治理”。
